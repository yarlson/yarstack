# journey-docs

Journey-docs creates concise Markdown docs-as-code artifacts for critical user journeys, including journey briefs, flows, acceptance scenarios, telemetry slices, service blueprints, portfolios, story maps, and coverage matrices.

## Known Limits

- Writes Markdown artifacts only; it does not create dashboards, tickets, or visual design files.
- Uses `TBD(owner)` when evidence or ownership is missing.
- Best for one persona and one goal per journey; broader requests should be split.

## Install

```bash
npx skills add https://github.com/yarlson/skills/tree/main/journey-docs
```
