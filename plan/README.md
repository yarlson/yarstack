# plan

Plan writes implementation plans that guide later code toward clean structure: evidence-backed phases, clear boundaries, concrete acceptance checks, and explicit traps to avoid before implementation starts.

## Known Limits

- Does not implement the plan or validate behavior by itself.
- Needs repo-local evidence for accurate phases; missing code, docs, or config should be called out.
- Plans should stay scoped to the requested work, not become a general roadmap.

## Install

```bash
npx skills add https://github.com/yarlson/skills/tree/main/plan
```
